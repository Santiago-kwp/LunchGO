# 리마인더 성능 및 안정성 개선 정리

본 문서는 리마인더 영역에서 수행한
성능 및 안정성 개선 작업을 정리한 문서이다.
각 항목은 적용 내용, 기대 효과, 측정 지표, 출처를 포함한다.

---

## 1. 프론트 트리거 제거 → 서버 스케줄러 전환

### 대상
- 리마인더 발송 로직

### 적용 내용
- 프론트 이벤트 기반 발송 구조 제거
- 서버 스케줄러에서 주기적으로 발송 대상 조회

### 기대 효과
- 사용자 브라우저 상태와 무관한 안정적 발송
- 프론트 요청 누락으로 인한 재시도 및 오류 제거

### 측정 지표
- 별도 수치 기록 없음 (발송 안정성 목적)

### 출처
- `docs/reminder-troubleshooting.md`

## 2. 리마인더 중복 발송 방지 상태 관리

### 대상
- 리마인더 발송 처리

### 적용 내용
- 리마인더 발송 여부를 DB에 기록
- 발송 전 상태 체크를 통해 중복 발송 차단

### 기대 효과
- 동일 예약에 대한 중복 SMS 발송 제거
- 불필요한 외부 메시지 비용 감소

### 측정 지표
- 별도 수치 기록 없음 (비용·중복 방지 목적)

### 출처
- `docs/reminder-troubleshooting.md`

---

## 3. 리마인더 링크 도메인 설정 분리

### 대상
- 리마인더 문자 링크 생성 로직

### 적용 내용
- 링크 도메인을 코드가 아닌 설정 값(`public-base-url`)으로 분리
- 로컬/배포 환경별 값 관리

### 기대 효과
- 환경 전환 시 코드 수정 및 재배포 감소
- 잘못된 링크로 인한 재요청 및 문의 비용 감소

### 측정 지표
- 별도 수치 기록 없음 (운영 안정성 목적)

### 출처
- `docs/reminder-troubleshooting.md`
