# 예약 조회 성능 최적화 정리

본 문서는 예약 조회 영역에서 수행한
성능 및 안정성 개선 작업을 정리한 문서이다.
각 항목은 적용 내용, 기대 효과, 측정 지표, 출처를 포함한다.

---

## 1. 예약 상태 기준 조회 분리

### 대상
- 사용자 예약 조회 API
- 점주 예약 조회 API

### 적용 내용
- 예약 상태(TEMPORARY / CONFIRMED / CANCELLED / EXPIRED 등)에 따라
  조회 대상을 명확히 분리

### 기대 효과
- 불필요한 예약 데이터 조회 감소
- 조회 결과 가공 로직 단순화

### 측정 지표
- 별도 수치 기록 없음 (쿼리 조건 명확화 목적)

### 출처
- 프로젝트 예약 조회 구현 내역

---

## 2. 결제 실패 예약 조회

### 대상
- 사용자 예약 내역 조회

### 적용 내용
- 결제 실패 또는 미결제 예약은 결제 대기로 표기

### 기대 효과
- 사용자 화면에서 불필요한 예약 데이터 제거
- 프론트 단 추가 분기 처리 감소

### 측정 지표
- 별도 수치 기록 없음

### 출처
- 예약 조회 관련 대화 및 구현 내역
