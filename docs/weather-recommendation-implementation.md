# 날씨 기반 식당 추천

## 목적
- 홈 화면에서 "날씨" 버튼 클릭 시, 현재 날씨에 맞는 음식점을 우선 추천한다.

## 데이터 소스
- 현재 날씨: OpenWeather API (`/api/weather/current`)
- 식당 정보:
  - 식당 이름, 설명
  - 메뉴명, 메뉴설명
  - 태그: `restaurantTags`, `tags`, `topTags`

## 날씨 분류 기준
- 비 올 때: `condition`이 `Rain`, `Drizzle`, `Thunderstorm`
- 눈 올 때: `condition`이 `Snow`
- 더울 때: `temp >= 28`
- 추울 때: `temp <= 7`
- 기타: 위 조건에 해당하지 않는 경우

## 추천 키워드 매핑
- 비 올 때: `국물`, `찌개`, `전골`, `칼국수`, `우동`, `짬뽕`, `수제비`
- 눈 올 때: `국물`, `탕`, `전골`, `보양`, `뜨끈`, `곰탕`, `설렁탕`, `갈비탕`
- 더울 때: `냉면`, `소바`, `샐러드`, `회`, `초밥`, `시원`
- 추울 때: `국`, `탕`, `찌개`, `전골`, `칼국수`, `우동`, `라면`, `뜨끈`, `따뜻`
- 기타: `가벼운`, `정식`, `한식`, `분식`, `파스타`, `덮밥`, `샌드위치`

## 매칭 방식(스코어링)
- 식당의 태그에 키워드가 포함되면 +2점
- 식당 텍스트(이름/설명/메뉴명/메뉴설명)에 키워드가 포함되면 +1점
- 점수가 높은 순으로 추천 정렬

## API
- `GET /api/weather/current?lat={lat}&lon={lon}`
- 응답 예시:
  ```json
  {
    "temp": 23.1,
    "feelsLike": 22.5,
    "condition": "Rain",
    "description": "light rain"
  }
  ```

## 예외 처리
- 날씨 API 실패 시 추천 리스트는 비움
